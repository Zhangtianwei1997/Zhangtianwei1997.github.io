如何将图片上传到Hexo博客上呢

2020-03-22

图片导入博客

环境安装

  1. 将主页配置文件_config.yml 里的 post_asset_folder: 这个选项设置为true
  2. 在 hexo 目录下执行 npm install hexo-asset-image –save
    这是下载安装一个可以上传本地图片的插件
  3. 等待一小段时间后，再运行hexo n “xxxx”来生成md博文时，/source/_posts 文件夹内除了xxxx.md文件还有一个同名的文件夹

导入图片

  1. 在xxxx.md中想引入图片时，先把图片复制到xxxx这个文件夹中，然后只需要在xxxx.md中按照markdown的格式引入图片：![你想输入的替代文字](xxxx/图片名.jpg)

示例：

出现啦！

  1. 第二种导入方法。

    示例

    用此种方法，而不是以前的方法，前提是你的hexo的版本是hexo3以上，到package.json里面看一下吧。如果不是hexo3以上的版本，那就只能用第二种方法了！

  2. 为什么有的人无法输出图片呢，会报错？

    • 没有在 blog (就是你运行 hexo 的地址) 下安装图片插件 ————》 重新安装
    • 检查你的插件版本，使用 npm install https://github.com/CodeFalling/hexo-asset-image –save 安装0.0.5版本的hexo-asset-image插件。 使用 npm install hexo-asset-image –save 安装的是1.0.0版本的hexo-asset-image插件。 两者最直接的区别是映射关系不同。 具体详情见 http://www.99708.com.cn/2019/07/26/hexo-本地图片不显示 ——————》重新安装就可

hexo博客搭建

2020-03-18

Windows 常用 cmd 指令

windows 下在地址栏输入 cmd 直接打开 command 地址为当前

linux —–> windows
pwd chdir
ls dir
cd cd

第一步 安装 node.js 与 git 环境

安装 node.js
网址——> node.js.org
Windows 建议下载 .msi 格式，下载完成无脑 “下一步” 0-0
顺便下载 git 并安装

第二步 更换 npm 源并安装 hexo

安装并更换 npm 源
npm install -g cnpm registry=https://registry.npm.taobao.org (马爸爸牛逼0-0)
回车，等

cnpm -v 回车看版本
cnpm install -g hexo-cli 回车,等
hexo -v 回车看版本

第三步 初始化第一个博客

建立一个 blog 文件夹
cd 到 blog
hexo init 直接初始化一个博客 （安装 git 后可以顺利进行） 等

第四步 查看 hello world 博客

输入 hexo s
localhost 4000 打开阅览器 输入 localhost：4000 进入
Ctrl + c 断开

第五步 编辑新的博客

hexo n “myblogname”
cd —> source/_posts
查看生成的博客
使用编辑软件打开就可以进行编辑
编辑完保存退出

第六步 重新生成

cd 到 blog 目录处
cd ../.. 返回两层目录
hexo g (generated)
hexo s —-> localhost 4000 —–> F5 一下 就有新的博客了
断开博客 ctrl + c

第七步 部署到远端

登录 Github （快去注册一个呀）
右上角 + 号点击创建仓库
名称 yournameid.github.io (Zhangtianwei1997.github.io)
回到命令行，在 blog 下输入
cnpm install hexo-deployer-git –save 等待安装完成
设置 ——config.yml
使用编辑器（notepad++ 或者 vscode 什么的）打开
在最后几行更改为：
deploy:
type: ‘git’
repo: https://github.com/Zhangtianwei1997/Zhangtianwei1997.github.io.git （你的 github 仓库地址）
branch: master
然后回到命令行
hexo d 部署到远端
报错是地址的，检查上面 repo 的网址前面有没有空格，注意 type 后面的单引号
who you are 使用
git config –global user.email “1349114380@qq.com“
git config –global user.name “Zhangtianwei1997”
设置全局变量 注意空格
再次 hexo d 部署
输入 Zhangtianwei1997.github.io.git 就可以进入你的博客了

第八步 更换主题

https://github.com/litten/hexo-theme-yilia 主题*1
在 blog 下载主题
git clone https://github.com/TryGhost/Casper.git themes/Casper
在 blog 文件下 themes/yilia 下便可以看见主题
修改 _config.yml 文件hex
theme: yilia （你的皮肤文件名称）
保存退出即可
hexo g 更新
hexo s 部署
hexo d 推送

额外 gitee部署

基本与 Github 一致，只是部署到 Gitee 上去
npm install hexo-asset-image –save
修改 ——config.yml
post_asset_folder: true
位置在 test/source/_posts下，会同时生成test目录和test.md 网页，将需要的图片放置到test目录下，test.md就可以正常解析了，注意：要在图片路径直接写成同目录下即可。
